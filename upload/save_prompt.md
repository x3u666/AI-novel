# Промпт: Переработка системы сохранений

## Контекст проекта

Next.js 16 + React 19 + TypeScript + Zustand + localStorage. Текущие файлы системы сохранений:
- `src/services/saveService.ts` — работа с localStorage (4 слота: 0 авто + 1-3 ручные)
- `src/stores/gameStore.ts` — Zustand стор, методы `loadGame`, `saveGame`, `setEnding`
- `src/types/game.ts` — интерфейсы `SaveSlot`, `GameState`
- `src/components/modals/SaveGameModal.tsx` — модалка ручного сохранения в игре
- `src/components/modals/LoadGameModal.tsx` — модалка загрузки из главного меню
- `src/hooks/useAutoSave.ts` — автосохранение по таймеру и после решений
- `src/app/page.tsx` — главное меню (кнопки «Новая игра», «Продолжить»)
- `src/app/select-narrator/page.tsx` — выбор нарратора
- `src/app/game/page.tsx` — игровой экран

## Задача

Полностью переработать систему сохранений. Ниже описана новая логика **по шагам**.

---

## 1. Новый флоу «Новая игра»

**Было:** Новая игра → выбор нарратора → игра  
**Стало:** Новая игра → **выбор слота** → выбор нарратора → игра

### Детали

При нажатии «Новая игра» в главном меню (`src/app/page.tsx`) открывается **модальное окно выбора слота** (`NewGameSlotModal`).

Окно показывает **3 слота** (индексы 1, 2, 3 — без слота 0, он зарезервирован под автосохранение):

```
┌─────────────────────────────────────┐
│  Выберите слот для сохранения        │
├─────────────────────────────────────┤
│  Слот 1  │  [пусто]                 │  ← пустой слот
│  Слот 2  │  Глава 3 · 01:24 · дата  │  ← занятый (показать превью)
│  Слот 3  │  Глава 1 · 00:08 · дата  │  ← занятый
├─────────────────────────────────────┤
│  При выборе занятого слота —         │
│  показать предупреждение о           │
│  перезаписи с кнопками              │
│  [Отмена] [Перезаписать]            │
└─────────────────────────────────────┘
```

**Логика:**
- Пустой слот → сразу переходит к выбору нарратора, слот записывается в стор (`selectedSlotIndex`)
- Занятый слот → показывает inline-предупреждение о перезаписи → при подтверждении удаляет старые данные, записывает слот в стор, переходит к выбору нарратора
- Кнопка «Назад» закрывает модалку без действий

**Что добавить в `gameStore`:**
```typescript
selectedSlotIndex: number | null  // какой слот выбран для текущей игры
setSelectedSlot: (index: number) => void
```

## 2. Автосохранение — новая логика

**Слот 0** зарезервирован как «последняя активная игра» (не отображается пользователю как отдельный слот).

### Когда срабатывает автосохранение:
1. **По таймеру** — каждые 3 минуты, если игра активна (`isGameStarted && !isFinished`)
2. **После каждого решения** — с задержкой 500мс после `makeChoice`
3. **При выходе из игры** — при нажатии кнопки «Домой» (до перехода на главное меню) — сохраняется и в слот 0 (автосейв), и в `selectedSlotIndex`
4. **При концовке** — при вызове `setEnding` немедленно сохраняется в оба слота (слот 0 и `selectedSlotIndex`) с флагами `isFinished: true`, `endingId`

### Что убрать:
- Убрать слот 0 из UI (не показывать в `LoadGameModal`, `SaveGameModal`)
- Убрать `SaveGameModal` из игрового экрана — ручное сохранение в слот происходит автоматически при выходе, пользователь не выбирает слот во время игры
- Убрать кнопку «Сохранить» (💾) из тулбара игрового экрана

---

## 3. Кнопка «Продолжить» в главном меню

- Активна, если слот 0 (автосейв) не пуст
- При нажатии — загружает слот 0 напрямую, без модалки
- Переходит на `/game`
- Если игра в слоте 0 завершена (`isFinished: true`) — всё равно загружает, но игровой экран работает в **режиме просмотра** (см. п. 5)

---

## 4. Модалка «Загрузить игру» в главном меню

Показывает **слоты 1, 2, 3** (не слот 0).

Для каждого слота:
- Название слота (Слот 1/2/3)
- Имя нарратора
- Глава
- Время прохождения
- Дата сохранения
- Превью текста (первые 80 символов последнего нарративного блока)
- Если `isFinished: true` — бейдж «Завершено» с цветом концовки (зелёный/серый/красный)
- Кнопка 🗑️ удалить слот (с подтверждением)
- Пустой слот — показывается как пустая карточка с пунктирной рамкой «Пусто»

При клике на занятый слот → загружает игру → переходит на `/game`.

---

## 5. Режим просмотра завершённой игры

Если загружается слот с `isFinished: true`:

**Что работает:**
- Листать историю сообщений в чате (скролл)
- Листать нарративную панель
- Открывать Дневник, Карту решений, Персонажей

**Что заблокировано:**
- Поле ввода сообщений — скрыто полностью (не задизейблено, а убрано из DOM)
- Кнопки выбора действий — скрыты полностью
- Автосохранение — не срабатывает (`enabled: !isFinished`)

**Плашка вверху чата** (вместо поля ввода):
```
┌──────────────────────────────────────┐
│  ✦  История завершена                │
│     Это просмотр пройденной игры     │
└──────────────────────────────────────┘
```
Стиль: полупрозрачный фон, акцентный цвет нарратора, шрифт Inter.

**При загрузке завершённой игры** — никакой инициализации (`initializeGame` не вызывается), история сообщений отображается как есть из `chatHistory`.

## 6. Перезапись слота

Перезапись возможна только через `NewGameSlotModal` при старте новой игры:
- Пользователь выбирает уже занятый слот
- Появляется предупреждение прямо в модалке (не отдельный диалог)
- Подтверждение → `deleteSlot(index)` → `setSelectedSlot(index)` → переход к нарратору

Во время игры перезаписи нет — всё сохраняется в заранее выбранный слот автоматически.

## 7. Изменения в `saveService.ts`

```typescript
// Новые/изменённые функции:

// Автосейв — всегда слот 0
saveAutoSlot(gameState: GameState): void

// Сохранение в выбранный слот пользователя (1-3)
saveToSelectedSlot(slotIndex: number, gameState: GameState): void

// Сохранение в оба места одновременно
saveGame(slotIndex: number, gameState: GameState): void {
  saveAutoSlot(gameState)       // слот 0
  saveToSelectedSlot(slotIndex, gameState)  // слот 1-3
}

// Загрузить автосейв (для «Продолжить»)
loadAutoSave(): SaveSlot | null  // loadSlot(0)

// Проверить есть ли автосейв
hasAutoSave(): boolean
```

## Стиль и UX

- Все модалки в едином стиле: `bg-[#1a1a24]`, `border-white/10`, тёмная тема
- Анимации через Framer Motion (fade + scale)
- Бейдж «Завершено» использует цвет концовки из `endings.ts` (`good` → зелёный, `neutral` → серый, `bad` → красный)
- Предупреждение о перезаписи — inline внутри модалки, не отдельный `Dialog`
- Пустые слоты — пунктирная рамка `border-dashed border-white/20`
