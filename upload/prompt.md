

# AI-Платформа создания визуальной новеллы — Расширенный гайд (Веб-приложение)

> **Область фокуса:** Полное описание фронтенда, UI/UX, пользовательских сценариев, структуры данных и взаимодействий — **без LLM/ML**, но с заделом на будущую интеграцию. Все заглушки и моковые данные описаны явно.

---

## ЧАСТЬ 1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

### 1.1 Цель текущего этапа

Создать полностью функциональное веб-приложение визуальной новеллы со всеми экранами, навигацией, системой сохранений, настройками, выбором пресетов и игровым интерфейсом. На данном этапе вместо реального LLM используются **заглушки (mock-данные)**: заранее написанные нарративные блоки, варианты выбора и сюжетные ветки. Архитектура фронтенда должна быть спроектирована так, чтобы замена моковых данных на реальные ответы LLM потребовала минимальных изменений.

### 1.2 Принцип «LLM-ready» архитектуры

- Все функции, которые в будущем будут обращаться к LLM, оформлены как **асинхронные сервисные функции** с единым интерфейсом (например, `generateNarrative(context): Promise<NarrativeResponse>`).
- Сервисный слой (service layer) изолирован: на текущем этапе сервис возвращает mock-данные, в будущем — ответ от API бэкенда.
- Все структуры данных (игровое состояние, история решений, entity memory) уже содержат полные поля, необходимые для передачи контекста в LLM.
- Typing-индикатор и эффект посимвольного появления текста реализуются уже сейчас (на моковых данных с искусственной задержкой).

## ЧАСТЬ 2. ТЕХНИЧЕСКИЙ СТЕК (Фронтенд-фаза)

### 2.1 Используемые технологии

| Технология | Назначение | Обоснование |
|---|---|---|
| **React 18+** (или Next.js 14+ с App Router) | Основной фреймворк | Компонентная архитектура, серверные компоненты (Next.js) для SSR |
| **TypeScript** | Типизация | Строгие типы для игрового состояния, пресетов, сохранений |
| **Tailwind CSS** | Стилизация | Утилитарный подход, быстрая кастомизация тёмной/светлой темы |
| **Framer Motion** | Анимации | Плавные переходы экранов, появление текста, hover-эффекты карточек |
| **Zustand** | Управление состоянием | Лёгкий, без boilerplate, поддержка persist-middleware для localStorage |
| **React Router v6** (если не Next.js) | Маршрутизация | Навигация между экранами |
| **Lucide React** или **Heroicons** | Иконки | Консистентная иконография для toolbar |
| **Google Fonts** (Playfair Display + Inter) | Типографика | Serif для нарратива, sans-serif для UI |
| **localStorage** | Хранение сохранений | До 4 слотов, без сервера на этапе MVP |
| **Web Audio API** (опционально) | Фоновая музыка | Атмосферный амбиент, зависящий от пресета |

### 2.3 Ключевые типы данных (TypeScript)

```typescript
// types/narrator.ts
type PresetId = 
  | 'neutral' 
  | 'knight' 
  | 'romantic' 
  | 'fighter' 
  | 'scientist' 
  | 'dark_mage' 
  | 'detective' 
  | 'horror'
  | 'poet'        // доп. пресет
  | 'comedian';   // доп. пресет

interface NarratorPreset {
  id: PresetId;
  name: string;                    // «Рыцарь», «Романтик» и т.д.
  subtitle: string;                // «Приключения», «Любовная история» и т.д.
  description: string;             // Развёрнутое описание стиля (2–3 абзаца)
  sampleText: string;              // Пример фрагмента текста в данном стиле
  avatarUrl: string;               // Путь к иллюстрации/аватару
  accentColor: string;             // Акцентный цвет пресета (hex)
  backgroundMusicUrl?: string;     // Путь к аудиофайлу амбиента
  tags: string[];                  // Теги: ['средневековье', 'эпос', 'героизм']
  isDefault: boolean;              // true только для 'neutral'
}

// types/game.ts
interface GameState {
  id: string;                      // Уникальный ID сессии
  presetId: PresetId;              // Выбранный пресет
  currentChapter: number;          // Номер текущей главы/сцены
  currentSceneId: string;          // ID текущей сцены в сюжетном дереве
  narrativeHistory: NarrativeBlock[]; // Все нарративные блоки (левая панель)
  chatHistory: ChatMessage[];      // Все сообщения чата (правая панель)
  decisions: Decision[];           // Все принятые решения
  characters: Character[];         // Встреченные персонажи
  worldFlags: Record<string, boolean | string | number>; // Флаги состояния мира
  entityMemory: EntityMemory;      // Сущности для контекста (задел под LLM)
  startedAt: string;               // ISO timestamp начала
  lastPlayedAt: string;            // ISO timestamp последней активности
  totalPlayTimeSeconds: number;    // Общее время игры в секундах
  isFinished: boolean;             // Достигнута ли концовка
  endingId?: string;               // ID концовки, если достигнута
}

interface NarrativeBlock {
  id: string;
  chapterNumber: number;
  chapterTitle?: string;           // «Глава 1: Начало пути»
  content: string;                 // Текст нарративного блока
  timestamp: string;               // Когда был сгенерирован
  relatedDecisionId?: string;      // Связь с решением, которое привело к блоку
}

interface ChatMessage {
  id: string;
  role: 'narrator' | 'user' | 'system'; // Кто отправил
  content: string;                 // Текст сообщения
  timestamp: string;
  choices?: Choice[];              // Варианты выбора (только для narrator)
  isInitialPrompt?: boolean;       // Первое сообщение рассказчика
}

interface Choice {
  id: string;
  text: string;                    // Текст варианта
  shortLabel?: string;             // Краткая метка для карты решений
  consequenceHint?: string;        // Подсказка о последствиях (опционально)
}

interface Decision {
  id: string;
  sceneId: string;                 // В какой сцене было принято
  choiceId: string;                // Какой вариант выбран
  choiceText: string;              // Текст выбранного варианта
  timestamp: string;
  chapterNumber: number;
}

interface Character {
  id: string;
  name: string;
  description: string;            // Краткое описание
  firstAppearedInChapter: number;
  relationship?: string;          // Отношение к герою: «союзник», «враг» и т.д.
  avatarUrl?: string;
}

interface EntityMemory {
  playerName?: string;
  currentLocation: string;
  knownCharacters: string[];
  activeQuests: string[];
  importantItems: string[];
  moralAlignment: 'good' | 'neutral' | 'evil'; // Сдвиг морали на основе решений
  customFlags: Record<string, any>;
}

interface SaveSlot {
  slotIndex: number;               // 0–3
  type: 'manual' | 'auto';
  customName?: string;             // Пользовательское имя сохранения
  gameState: GameState;
  savedAt: string;                 // ISO timestamp
  previewText: string;             // Первые 100 символов последнего нарратива
  presetName: string;              // Имя пресета для отображения
  chapterInfo: string;             // «Глава 2, сцена 5»
  playTime: string;               // Отформатированное время: «01:23:45»
  thumbnailColor: string;          // Цвет-маркер на основе пресета
}

// types/ui.ts
interface SettingsConfig {
  textSize: 'small' | 'medium' | 'large' | 'xlarge';
  typingSpeed: number;             // Символов в секунду (20–100)
  autoPlaySpeed: number;           // Задержка между блоками при авто-прокрутке (мс)
  musicVolume: number;             // 0–100
  sfxVolume: number;               // 0–100
  theme: 'dark' | 'light';
  language: 'ru' | 'en';          // Задел на мультиязычность
}

type ModalType = 
  | 'settings' 
  | 'save' 
  | 'load' 
  | 'confirmQuit' 
  | 'diary' 
  | 'decisionMap' 
  | 'characters' 
  | 'help'
  | null;
```

#### Фон главного меню

- **Вариант A (предпочтительный):** Статичное атмосферное изображение с лёгким эффектом параллакса при движении мыши. Изображение — абстрактное или тематическое (раскрытая книга, звёздное небо, перо и чернила). Поверх изображения — полупрозрачный тёмный градиент (сверху и снизу), чтобы текст и кнопки были читабельны.
- **Вариант B:** CSS-анимированный градиентный фон с медленно перемещающимися мягкими цветовыми пятнами (blobs). Цвета — тёмные оттенки фиолетового, синего, бордового.
- **Vignette-эффект:** По краям экрана — затемнение (CSS radial-gradient), фокусирующее внимание на центре.

#### Логотип / Название

- **Расположение:** Верхняя треть экрана, по горизонтальному центру.
- **Стиль:** Шрифт Playfair Display, Bold, 36–44px. Цвет — основной текст с лёгким золотистым оттенком или градиентом.

- **Анимация появления:** Кнопки появляются последовательно (stagger 0.1s) с эффектом slide-up + fade-in, начиная с задержки 1.2s после загрузки страницы.

- Всегда активна.

```
│                          │  └─────────────────────────────┘      │
│                          │                                       │
│                          │  ┌───────────────────────┐ [Отпр.]   │
│                          │  │ Или напишите своё...   │            │
│                          │  └───────────────────────┘            │
│                          │                              [Авто]   │
├──────────────────────────┴───────────────────────────────────────┤
```

**Пропорции панелей:**
- Левая панель: **40%** ширины экрана.
- Правая панель: **60%** ширины экрана.
- Разделитель: Тонкая вертикальная линия (`1px`, цвет разделителя). Опционально — перетаскиваемая (drag-to-resize), но это не приоритет для MVP.

**Высота:** Обе панели занимают всё пространство от toolbar до нижнего края окна (`calc(100vh - toolbar_height)`).

#### Toolbar на игровом экране

Аналогичен Экрану 2, но с дополнениями:

- **По центру:** Индикатор пресета (`🟡 Рыцарь`) + номер текущей главы (`Глава 2`). При клике на номер главы — ничего (статичная информация).
- **Дополнительные иконки справа:**
  - **📖 Дневник** — открывает панель «Дневник героя».
  - **👥 Персонажи** — открывает панель «Персонажи» (если есть встреченные персонажи).
  - **🗺️ Карта решений** — открывает визуализацию дерева решений.
  - **⚙️ Настройки** — модалка настроек.
  - **💾 Сохранить** — модалка сохранения (слоты в режиме записи).
  - **🏠 Меню** — возврат в главное меню (с подтверждением).

Если иконок много — на узких экранах они сворачиваются в выпадающее меню (⋮ три точки).

#### Левая панель — Нарративная область

**Функция:** Постепенно накапливающийся текст-повествование. Здесь отображается «книга» — литературное описание событий, основанное на том, что происходит в чате.

**Структура содержимого:**

Каждый нарративный блок (NarrativeBlock) отображается как отдельная секция:

```
┌──────────────────────────┐
│                          │
│  Глава 1: Начало пути    │  ← Заголовок главы, Playfair Display, Bold, 22px,
│  ═════════════════════   │     акцентный цвет. Отображается только при смене главы.
│                          │
│  Солнце медленно         │  ← Текст нарратива, Playfair Display, Regular,
│  садилось за горизонт.   │     размер по настройкам. Межстрочный интервал 1.7.
│  Последние лучи          │
│  окрашивали каменные     │
│  стены деревни в         │
│  тёплый оранжевый цвет.  │
│                          │
│  · · ·                   │  ← Визуальный разделитель между блоками:
│                          │     три точки, или тонкая линия, или орнамент.
│                          │
│  Вы решили свернуть      │  ← Следующий блок нарратива. Если связан с решением —
│  на тропу через лес.     │     может быть выделен (лёгкий акцентный фон или
│  Деревья смыкались...    │     вертикальная полоска).
│                          │
└──────────────────────────┘
```

**Визуальные детали:**
- **Отступы:** padding 24px по горизонтали, 16px по вертикали.
- **Заголовки глав:** Playfair Display, Bold, 22px, акцентный цвет пресета. Верхний отступ: 32px (кроме первой главы). Под заголовком — декоративная линия (тонкая, акцентный цвет, ширина 60px, opacity 0.5).
- **Разделители блоков:** Три декоративные точки (`· · ·`) по центру, вторичный цвет, отступ сверху и снизу по 16px. Или тонкая горизонтальная линия (ширина 40%, по центру, цвет разделителя).
- **Блоки, связанные с решениями:** Могут иметь тонкую вертикальную полоску слева (3px, акцентный цвет, opacity 0.3) как маркер «ваш выбор привёл сюда».

**Поведение:**
- **Прокрутка:** Overflow-y: auto. Полоса прокрутки — стилизованная (тонкая, полупрозрачная, scrollbar-track прозрачный).
- **Автоматическая прокрутка вниз:** При появлении нового блока нарратива — плавная прокрутка к нему (`scrollIntoView({ behavior: 'smooth' })`).
- **Перечитывание:** Пользователь может прокрутить вверх и перечитать любой предыдущий блок.
- **Появление нового блока:** Framer Motion, fade-in + slide-up (duration: 0.5s). Текст внутри блока **не** использует typewriter-эффект (он только в чате) — появляется сразу весь блок, но сам блок анимированно «проявляется».

**Пустое состояние (начало игры):**
- Текст: «История ещё не началась. Расскажите рассказчику, с чего начать, и нарратив появится здесь.»
- Шрифт Playfair Display, Italic, вторичный цвет текста, по центру вертикально и горизонтально.

#### Правая панель — Чат с рассказчиком

**Функция:** Интерфейс диалогового взаимодействия с рассказчиком (в будущем — с LLM). Здесь пользователь вводит свои описания, делает выборы, а рассказчик отвечает.

**Структура:**

```
┌─────────────────────────────────────────┐
│          Область сообщений               │  ← Scrollable
│  (чат-лента)                            │
│                                         │
│  ...сообщения...                        │
│                                         │
├─────────────────────────────────────────┤
│  [Кнопки выбора, если есть]            │  ← Фиксированные
├─────────────────────────────────────────┤
│  ┌─────────────────────────┐  [Отпр.]  │  ← Поле ввода
│  │ Напишите своё действие..│            │     фиксировано внизу
│  └─────────────────────────┘   [Авто]  │
└─────────────────────────────────────────┘
```

**Сообщения чата:**

Каждое сообщение (`ChatMessage`) отображается в стиле чат-пузыря:

**Сообщение рассказчика (role: 'narrator'):**
```
┌─────────────────────────────────────┐
│ 🧙 Рассказчик              15:30    │
│                                     │
│ Древний лес раскинулся перед        │
│ тобой. Столетние дубы переплели     │
│ свои ветви так плотно, что          │
│ солнечный свет едва пробивался...   │
└─────────────────────────────────────┘
```
- **Аватар:** Иконка или маленький аватар рассказчика (соответствует выбранному пресету), 28×28px, border-radius: 50%.
- **Метка «Рассказчик»:** Inter, SemiBold, 13px, акцентный цвет пресета.
- **Timestamp:** Inter, Regular, 11px, вторичный цвет, справа от метки.
- **Текст:** Playfair Display, Regular, размер из настроек. Цвет — основной текст.
- **Фон пузыря:** Вторичный фон (`#1A1A24`), border-radius: 12px (0 в верхнем левом углу), padding: 12px 16px.
- **Выравнивание:** Слева, макс. ширина 85% от панели.
- **Typewriter-эффект:** Текст появляется посимвольно с настроенной скоростью. Во время «печатания» в конце текста мигает курсор (`|`). После завершения — курсор исчезает.

**Сообщение пользователя (role: 'user'):**
```
                    ┌──────────────────────┐
                    │                 15:31 │
                    │ Я осторожно вхожу в   │
                    │ лес, держа руку на    │
                    │ рукояти меча.         │
                    └──────────────────────┘
```
- **Без аватара** (или маленькая иконка пользователя).
- **Фон пузыря:** Полупрозрачный акцентный цвет (`rgba(акцент, 0.15)`), border-radius: 12px (0 в верхнем правом).
- **Выравнивание:** Справа, макс. ширина 75%.
- **Текст:** Inter, Regular, 15px.

**Системное сообщение (role: 'system'):**
```
        ─── Новая глава: Тёмный лес ───
```
- По центру, без пузыря. Текст Inter, Medium, 12px, вторичный цвет. Декоративные линии по бокам.

**Typing-индикатор:**

Пока «рассказчик генерирует ответ» (на данном этапе — искусственная задержка 1–3 секунды перед показом mock-данных):
```
┌──────────────────┐
│ 🧙 Рассказчик     │
│ ●  ●  ●           │  ← Три пульсирующие точки
└──────────────────┘
```
- Три точки, каждая пульсирует (scale 1 → 1.3 → 1) с задержкой 0.15s между ними.
- Появляется с fade-in, исчезает с fade-out при получении ответа.

#### Кнопки выбора (Choice Buttons)

Когда рассказчик предлагает варианты действий, под его сообщением (или в фиксированной зоне над полем ввода) появляются кнопки:

```
┌─────────────────────────────────┐
│ ⚔️  Войти в лес, несмотря на    │
│     предупреждения              │
└─────────────────────────────────┘
┌─────────────────────────────────┐
│ 🏠  Вернуться в деревню и       │
│     подготовиться               │
└─────────────────────────────────┘
┌─────────────────────────────────┐
│ 👁️  Осмотреться по сторонам     │
│     прежде чем решить           │
└─────────────────────────────────┘
```

**Стиль каждой кнопки:**
- Фон: Третичный фон (`#252532`).
- Рамка: `1px solid rgba(255,255,255,0.1)`.
- Border-radius: 10px.
- Padding: 12px 16px.
- Текст: Inter, Regular, 14px, основной цвет.
- Эмодзи-иконка слева (опционально).
- Hover: Рамка — акцентный цвет, фон чуть ярче, курсор pointer.
- Появление: Stagger animation, каждая кнопка slide-up + fade-in с задержкой 0.1s.
- **Ширина:** 100% ширины чат-области.

**Поведение при выборе:**
1. Кнопка, по которой кликнули, подсвечивается акцентным цветом (фон заливается акцентом, текст — тёмный).
2. Остальные кнопки fade-out (0.3s).
3. Выбранный вариант появляется как сообщение пользователя в чате.
4. Кнопки скрываются.
5. Появляется typing-индикатор.
6. Через задержку — ответ рассказчика + обновление нарративной панели.

#### Поле ввода (User Input)

**Расположение:** Фиксировано внизу правой панели.

**Структура:**
```
┌─────────────────────────────────┐  ┌────┐
│ Опишите своё действие...        │  │ ➤  │
│                                 │  └────┘
└─────────────────────────────────┘
                                      [Авто]
```

- **Текстовое поле:** `<textarea>` с автоматическим расширением по высоте (от 1 до 4 строк). Фон: третичный. Рамка: `1px solid rgba(255,255,255,0.1)`. При фокусе — рамка акцентного цвета. Border-radius: 10px. Padding: 10px 14px.
- **Placeholder:** «Опишите своё действие или мысли...» — вторичный цвет, Italic.
- **Кнопка «Отправить» (➤):** Справа от поля ввода. Круглая или квадратная с border-radius. Фон — акцентный цвет. Иконка — стрелка вправо, белая. Активна только если поле не пустое. При пустом — приглушена.
- **Отправка:** По кнопке или по `Enter` (Shift+Enter — новая строка).
- **Во время генерации ответа:** Поле ввода disabled, кнопка — spinner/loading.

**Кнопка «Авто» (Auto-play):**
- Маленькая кнопка под или рядом с полем ввода.
- Текст: «▶ Авто» (неактивно) / «⏸ Авто» (активно).
- При включении — чат автоматически прокручивается, и если есть варианты выбора — таймер (из настроек autoPlaySpeed) отсчитывает до автоматического выбора первого варианта. Или: если нет вариантов — автоматически посылает пустой ввод (рассказчик продолжает сам).
- Визуальный индикатор: полоска прогресса под кнопками выбора, показывающая, сколько времени осталось до авто-выбора.

#### Первый вход в игру (первое сообщение)

Последовательность действий при переходе с Экрана 2 на Экран 3:

1. Экран загружается. Левая панель — пустая (с текстом-заглушкой). Правая панель — пустая.
2. Через 0.5s — typing-индикатор рассказчика.
3. Через 1.5s — первое сообщение рассказчика (typewriter-эффект):
   > «Приветствую тебя, путник. Я — твой рассказчик, и мне не терпится узнать историю, которую мы создадим вместе. Расскажи мне: кто ты? Где ты находишься? Что видишь вокруг? Опиши начало своего пути — и я продолжу.»
   
   (Текст зависит от выбранного пресета — для Рыцаря — средневековый тон, для Учёного — научный и т.д.)
4. Поле ввода становится активным. Фокус автоматически переходит на поле ввода.
5. Пользователь вводит описание (минимум 10 символов для активации кнопки «Отправить» — валидация).
6. После отправки — его текст появляется как сообщение пользователя.
7. Typing-индикатор.
8. Ответ рассказчика — вводный нарративный блок. Одновременно в левой панели появляется первый блок нарратива (Глава 1).
9. В конце ответа — первые кнопки выбора.

### 3.8 Система сохранений — подробная механика

#### Структура слотов

| Индекс | Тип | Описание |
|---|---|---|
| 0 | Автосохранение | Автоматически перезаписывается. 1 слот. |
| 1 | Ручное | Создаётся пользователем. |
| 2 | Ручное | Создаётся пользователем. |
| 3 | Ручное | Создаётся пользователем. |

#### Автосохранение

**Когда происходит:**
- После каждого сюжетного выбора (решения).
- При переходе к новой главе.
- Каждые 5 минут реального времени (если были изменения в gameState).

**Как работает:**
1. Текущий `GameState` сериализуется в JSON.
2. Создаётся объект `SaveSlot` с `type: 'auto'`, текущим timestamp, превью-текстом.
3. Сохраняется в localStorage под ключом `save_slot_0`.
4. В правом нижнем углу экрана на 2 секунды появляется тихое уведомление: «💾 Автосохранение» — маленький toast, fade-in + fade-out.

#### Ручное сохранение

**Как вызвать:**
1. Иконка 💾 в toolbar.
2. Открывается модалка «Сохранить игру» — аналогичная «Загрузить игру», но:
   - Занятые слоты кликабельны для **перезаписи** (с подтверждением: «Перезаписать сохранение?»).
   - Пустые слоты кликабельны для **записи**.
   - Автосохранение (слот 0) **нельзя** перезаписать вручную.

**При сохранении:**
1. Пользователь кликает на слот.
2. (Если слот занят) Подтверждение перезаписи.
3. Опциональное переименование: появляется поле ввода с предзаполненным именем (по умолчанию: «Сохранение [дата]»). Можно оставить или изменить.
4. Сохранение в localStorage.
5. Toast-уведомление: «✅ Игра сохранена».

#### Загрузка

1. Пользователь кликает на занятый слот в модалке «Загрузить».
2. Если текущая игра не сохранена — подтверждение: «Загрузить сохранение? Несохранённый прогресс будет потерян.»
3. `GameState` десериализуется из localStorage.
4. Zustand store обновляется.
5. Экран 3 перерисовывается: нарративная панель заполняется историей из `narrativeHistory`, чат — из `chatHistory`.
6. Прокрутка обеих панелей — в самый низ (к последнему сообщению).

#### localStorage-ключи

```
save_slot_0  →  JSON<SaveSlot>  (автосохранение)
save_slot_1  →  JSON<SaveSlot>  (ручное)
save_slot_2  →  JSON<SaveSlot>  (ручное)
save_slot_3  →  JSON<SaveSlot>  (ручное)
settings     →  JSON<SettingsConfig>
last_session →  string (ID последней сессии, для кнопки «Продолжить»)
```

#### Сервисный слой (заглушка)

```typescript
// services/narrativeService.ts

interface NarrativeRequest {
  currentSceneId: string;
  choiceId?: string;
  userInput?: string;
  gameState: GameState;
}

interface NarrativeResponse {
  narratorMessage: string;
  narrativeBlock: string;
  choices: Choice[];
  newCharacters?: Character[];
  worldFlagUpdates?: Record<string, any>;
  nextSceneId: string;
  chapterUpdate?: { number: number; title: string };
}

// Mock-реализация (будет заменена на fetch к API)
async function generateNarrative(request: NarrativeRequest): Promise<NarrativeResponse> {
  // Имитация задержки LLM (1–3 секунды)
  await new Promise(resolve => setTimeout(resolve, 1500 + Math.random() * 1500));
  
  const nextScene = getNextScene(request);  // Поиск по дереву mock-данных
  
  return {
    narratorMessage: nextScene.narratorMessage,
    narrativeBlock: nextScene.narrativeBlock,
    choices: nextScene.choices,
    newCharacters: nextScene.newCharacters,
    worldFlagUpdates: nextScene.worldFlagUpdates,
    nextSceneId: nextScene.id,
    chapterUpdate: nextScene.chapterTitle 
      ? { number: nextScene.chapterNumber, title: nextScene.chapterTitle }
      : undefined,
  };
}
```

**Важно:** Интерфейсы `NarrativeRequest` и `NarrativeResponse` уже содержат все поля, которые потребуются для реального LLM-запроса. При интеграции с бэкендом — функция `generateNarrative` просто заменит mock-логику на `fetch('/api/generate', { body: JSON.stringify(request) })`.

### 3.10 «Режим испытания» (Challenge Mode)

**Описание:** В определённых напряжённых сценах (боевые сцены, погони, критические моменты) включается режим ограниченного времени.

**Визуальные изменения:**
- Фон игрового экрана получает лёгкий красноватый оттенок (overlay `rgba(255, 50, 50, 0.05)`).
- Над кнопками выбора появляется **таймер-полоска**: горизонтальная полоса, заполненная акцентным цветом, которая плавно убывает слева направо.
- Время: 30 секунд (настраиваемо в mock-данных для каждой сцены).
- Если время вышло — автоматически выбирается случайный (или «наихудший») вариант. Сообщение: «Время вышло! Судьба сделала выбор за вас.»
- Пульсирующий эффект на рамке кнопок выбора (тонкое красное свечение).
- Тиканье (звуковой эффект, если звук включён).

**В mock-данных:** Сцена помечена флагом `isTimed: true, timeLimit: 30`.

**Шрифт в PDF:** Serif (Georgia или встроенный), размер 12pt. Заголовки глав — 16pt, bold. Поля: 2.5cm.

## ЧАСТЬ 6. СОСТОЯНИЯ ОШИБОК И ГРАНИЧНЫЕ СЛУЧАИ

### 6.1 Ошибки localStorage

| Ситуация | Поведение |
|---|---|
| localStorage недоступен (приватный режим) | При загрузке — предупреждение: «Сохранения недоступны в приватном режиме браузера. Игра не будет сохраняться.» Кнопка «Продолжить» скрыта. Игра работает, но без сохранений. |
| localStorage переполнен | При попытке сохранения — ошибка: «Недостаточно места для сохранения. Удалите старые сохранения.» |
| Повреждённые данные сохранения | При загрузке — если JSON.parse выбросил ошибку, слот помечается как «Повреждённое сохранение» с кнопкой «Удалить». |

### 6.2 Прочие граничные случаи

| Ситуация | Поведение |
|---|---|
| Пользователь вводит пустой текст | Кнопка «Отправить» неактивна. |
| Пользователь вводит > 500 символов | Счётчик символов. При превышении — текст обрезается или кнопка неактивна с подсказкой. |
| Пользователь обновляет страницу во время игры | Если есть автосохранение — при следующем входе «Продолжить» активна. Если нет — прогресс потерян. |
| Пользователь пытается начать новую игру с незавершённой | Подтверждение: «Начать новую игру? Текущий прогресс будет потерян, если не сохранён.» |

## ЧАСТЬ 8. ПРЕДЛОЖЕНИЯ ПО ДОПОЛНЕНИЮ ДИЗАЙНА И ФУНКЦИОНАЛА

> Ниже — мои собственные идеи для улучшения проекта на уровне веб-приложения (без LLM/ML).

### 8.1 Дизайн

1. **Динамическая тема по пресету.** При выборе пресета не только акцентный цвет меняется, но и весь визуальный стиль: фоновая текстура (пергамент для Рыцаря, металлический блеск для Бойца, сетка кода для Учёного), шрифт нарративного текста (можно заменять на тематический — например, готический для Хоррора), стиль разделителей между главами (мечи для Рыцаря, сердца для Романтика, шестерёнки для Учёного).

2. **Эффект «книжных страниц».** В левой нарративной панели при переходе к новой главе — анимация перелистывания страницы (CSS 3D transform). Или: каждая глава оформлена как «страница книги» с декоративной рамкой.

3. **Mood-индикатор.** Небольшой визуальный элемент (иконка + цвет) в углу экрана, показывающий текущее «настроение» истории: ⚔️ Опасность, 💕 Романтика, 🔍 Исследование, 😨 Страх. Меняется в зависимости от текущей сцены (задаётся в mock-данных). Создаёт дополнительную атмосферу.

4. **Анимация фона игрового экрана.** За панелями — едва заметный анимированный фон, зависящий от пресета: медленно летящие частицы (снег для зимних сцен, пыль для подземелий), лёгкое мерцание, смена цвета по мере развития сюжета.

5. **Splash-screen при запуске.** Кратковременный (1.5s) экран с логотипом и анимированной загрузкой перед показом главного меню. Задаёт первое впечатление.

6. **Уведомления-тосты.** Единая система toast-уведомлений (снизу-справа): автосохранение, успешное сохранение, новый персонаж встречен, достижение получено. Стек из максимум 3 тостов, каждый исчезает через 3s.
